# Homepage Gallery Rail Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the 1029px auto-crossfading slideshow under WHO ARE WE with a ~375px full-bleed rail of captioned, clickable landscape cards that drifts continuously and can be paused.

**Architecture:** Purely additive layers on a buildless static site, following the pattern the hero work established. The section wrapper `<section id="zOl98u">` is kept and its builder-generated contents are replaced wholesale. Styling goes in a new `assets/css/rail.css`, loaded only by `index.html`. The only JavaScript is a pause toggle in `assets/js/rail.js`; the drift itself is a CSS keyframe with no JS in its path. Downscaled image derivatives are generated by a committed shell script so the step is reproducible rather than a one-off.

**Tech Stack:** Plain HTML, CSS, vanilla ES5-compatible JS. `sips` (macOS built-in) for image derivatives. No build step, no bundler, no dependencies.

**Spec:** `docs/superpowers/specs/2026-07-19-gallery-rail-design.md`

**Branch:** `hero-recompose`. Do not push. Do not merge to `master`.

---

## Why there are no unit tests

This repo has no frontend test infrastructure and no `package.json`. Its only automated tests are `tests/chat.test.mjs` (chatbot, vitest) and `tools/knowledge/test_extract.py`. Adding Playwright would introduce `node_modules` to a site whose entire deployment model is copying files onto shared hosting.

The established pattern here is `tools/hero-check.js`: a dependency-free browser-console assertion script, written first, failing first, passing at the end. Task 1 builds the equivalent `tools/rail-check.js`. **That is the TDD loop for this work.** Do not skip Task 1 and do not implement anything before it fails for the right reason.

Read `tools/hero-check.js` before starting — Task 1 reuses its `check()` harness shape deliberately, so the two scripts stay recognisably siblings.

## Task 1: Rail assertion harness

- [ ] **Step 1: Write the assertion script**

Create `tools/rail-check.js`:

```js
/* UGCC gallery rail checks. Dependency-free.
   Usage: serve the site over HTTP, open the homepage, paste into the console.
   Returns a Promise of {passed, failed, results} — several checks fetch().

   Resize to 1280x720 before running the height check. Checks 5 and 7 are run
   separately at other viewports / emulated media, see the plan's Task 7.

   NOT covered (manual verification required):
     - whether the drift speed feels right
     - caption legibility over each individual photograph
     - touch drag behaviour on a real device */
(function () {
  'use strict';

  var results = [];
  function check(name, fn) {
    var ok = false, detail = '';
    try {
      var r = fn();
      if (r === true) { ok = true; }
      else if (r && typeof r === 'object') { ok = !!r.ok; detail = r.detail || ''; }
    } catch (e) { detail = 'threw: ' + e.message; }
    results.push({ name: name, ok: ok, detail: detail });
  }

  var SECTION = 'zOl98u';
  var EXPECTED_HREFS = [
    '/oil-and-gas-completed',
    '/civil-completed',
    '/water-completed',
    '/building-construction-completed',
    '/roads-and-bridges-completed'
  ];

  function railCards(all) {
    var sel = all ? '.v2-rail__item' : '.v2-rail__item:not([aria-hidden="true"])';
    return Array.prototype.slice.call(document.querySelectorAll(sel));
  }

  check('section is under 500px tall', function () {
    var s = document.getElementById(SECTION);
    if (!s) return { ok: false, detail: 'section #' + SECTION + ' missing' };
    var h = Math.round(s.getBoundingClientRect().height);
    if (h === 0) return { ok: false, detail: 'section has zero height — not rendered' };
    return { ok: h < 500, detail: h + 'px (was 1029px)' };
  });

  check('old slideshow is gone', function () {
    var n = document.querySelectorAll('.slideshow, .slide, .slideshow__dots').length;
    return { ok: n === 0, detail: n + ' legacy slideshow nodes remain' };
  });

  check('track holds 30 items, 15 of them aria-hidden', function () {
    var all = railCards(true).length;
    var hidden = railCards(true).filter(function (li) {
      return li.getAttribute('aria-hidden') === 'true';
    }).length;
    return {
      ok: all === 30 && hidden === 15,
      detail: all + ' items, ' + hidden + ' aria-hidden (want 30 / 15)'
    };
  });

  check('duplicate set is not keyboard reachable', function () {
    var bad = railCards(true).filter(function (li) {
      if (li.getAttribute('aria-hidden') !== 'true') return false;
      var a = li.querySelector('a');
      return !a || a.getAttribute('tabindex') !== '-1';
    }).length;
    return { ok: bad === 0, detail: bad + ' aria-hidden cards lack tabindex="-1"' };
  });

  check('every visible card has alt text and intrinsic size', function () {
    var bad = [];
    railCards(false).forEach(function (li, i) {
      var img = li.querySelector('img');
      if (!img) { bad.push('#' + i + ' no img'); return; }
      if (!img.getAttribute('alt')) bad.push('#' + i + ' empty alt');
      if (!img.getAttribute('width') || !img.getAttribute('height')) bad.push('#' + i + ' no width/height');
      if (img.getAttribute('loading') !== 'lazy') bad.push('#' + i + ' not lazy');
    });
    return { ok: bad.length === 0, detail: bad.join('; ') || '15/15 ok' };
  });

  check('no horizontal page overflow', function () {
    var over = document.body.scrollWidth - window.innerWidth;
    return {
      ok: over <= 0,
      detail: 'body.scrollWidth ' + document.body.scrollWidth +
              ' vs innerWidth ' + window.innerWidth + ' @ ' + window.innerWidth + 'px'
    };
  });

  var reduced = window.matchMedia('(prefers-reduced-motion: reduce)').matches;

  check('motion matches the reduced-motion preference', function () {
    var track = document.querySelector('.v2-rail__track');
    var vp = document.querySelector('.v2-rail__viewport');
    if (!track || !vp) return { ok: false, detail: 'rail not implemented' };
    var anim = getComputedStyle(track).animationName;
    var ovx = getComputedStyle(vp).overflowX;
    var branch = reduced ? 'reduce' : 'no-preference';
    if (reduced) {
      /* Under reduce the animation must actually be off AND the rail must stay
         browsable — an animation-only check would pass on a frozen, unreachable
         rail, which is worse than the slideshow it replaced. */
      return {
        ok: anim === 'none' && ovx === 'auto',
        detail: '[' + branch + '] animation-name=' + anim + '; overflow-x=' + ovx
      };
    }
    return {
      ok: anim !== 'none',
      detail: '[' + branch + '] animation-name=' + anim
    };
  });

  check('pause toggle controls the animation', function () {
    var btn = document.querySelector('.v2-rail__toggle');
    var track = document.querySelector('.v2-rail__track');
    if (reduced) {
      var hiddenOk = !btn || getComputedStyle(btn).display === 'none';
      return { ok: hiddenOk, detail: '[reduce] toggle hidden=' + hiddenOk };
    }
    if (!btn || !track) return { ok: false, detail: 'toggle or track missing' };
    var before = getComputedStyle(track).animationPlayState;
    btn.click();
    var pressed = btn.getAttribute('aria-pressed');
    var during = getComputedStyle(track).animationPlayState;
    btn.click();
    var after = getComputedStyle(track).animationPlayState;
    return {
      ok: before === 'running' && pressed === 'true' && during === 'paused' && after === 'running',
      detail: before + ' -> ' + during + ' -> ' + after + '; aria-pressed=' + pressed
    };
  });

  /* Async: every distinct href must resolve to a real page. */
  var hrefs = railCards(false).map(function (li) {
    var a = li.querySelector('a');
    return a ? a.getAttribute('href') : null;
  });
  var distinct = hrefs.filter(function (h, i) { return h && hrefs.indexOf(h) === i; });

  check('cards produce exactly the 5 expected hrefs', function () {
    var missing = EXPECTED_HREFS.filter(function (h) { return distinct.indexOf(h) === -1; });
    var extra = distinct.filter(function (h) { return EXPECTED_HREFS.indexOf(h) === -1; });
    return {
      ok: missing.length === 0 && extra.length === 0,
      detail: 'missing[' + missing.join(',') + '] extra[' + extra.join(',') + ']'
    };
  });

  return Promise.all(distinct.map(function (h) {
    return fetch(h, { method: 'GET' })
      .then(function (r) { return { h: h, ok: r.ok }; })
      .catch(function () { return { h: h, ok: false }; });
  })).then(function (linkResults) {
    var dead = linkResults.filter(function (r) { return !r.ok; }).map(function (r) { return r.h; });
    results.push({
      name: 'every href resolves',
      ok: dead.length === 0 && linkResults.length > 0,
      detail: dead.length ? 'dead: ' + dead.join(', ') : linkResults.length + '/' + linkResults.length + ' ok'
    });

    var passed = results.filter(function (r) { return r.ok; }).length;
    var failed = results.length - passed;
    results.forEach(function (r) {
      console.log((r.ok ? 'PASS  ' : 'FAIL  ') + r.name + (r.detail ? '  [' + r.detail + ']' : ''));
    });
    console.log('\n' + passed + ' passed, ' + failed + ' failed');
    return { passed: passed, failed: failed, results: results };
  });
})();
```

## Task 2: Image derivatives

**Files:**
- Create: `tools/make-rail-images.sh`
- Create: `assets/img/v2/rail/slide-01-480.jpg` … `slide-15-960.jpg` (30 files)

Cards render at 352px CSS width and a 3:2-ish aspect. Derivatives are cropped to exactly 480x316 and 960x632 so every `<img>` can carry the same `width`/`height` attributes and reserve identical layout space. The source images are **not** a uniform aspect ratio (they range from 1920x1043 to 1920x1438), which is why the script picks the resample axis per image before cropping.

- [ ] **Step 1: Write the generation script**

Create `tools/make-rail-images.sh`:

```bash
#!/usr/bin/env bash
# Generates the homepage rail's downscaled derivatives from the 1920w originals.
# Idempotent: safe to re-run. Requires macOS `sips` (no external dependencies).
#
# Output: assets/img/v2/rail/slide-NN-{480,960}.jpg, centre-cropped to an exact
# 480x316 / 960x632 so every card reserves the same layout space.
set -euo pipefail

cd "$(dirname "$0")/.."
SRC="assets/img/v2"
OUT="assets/img/v2/rail"
mkdir -p "$OUT"

make_one() {
  local src="$1" out="$2" tw="$3" th="$4" w h
  read -r w h < <(sips -g pixelWidth -g pixelHeight "$src" \
    | awk '/pixelWidth/{w=$2} /pixelHeight/{h=$2} END{print w, h}')

  # Resample along whichever axis leaves both dimensions >= the target,
  # then centre-crop. Cropping without this can letterbox instead of crop.
  if [ $(( w * th )) -gt $(( h * tw )) ]; then
    sips --resampleHeight "$th" "$src" --out "$out" >/dev/null
  else
    sips --resampleWidth "$tw" "$src" --out "$out" >/dev/null
  fi
  sips -c "$th" "$tw" "$out" >/dev/null
}

for n in 01 02 03 04 05 06 07 08 09 10 11 12 13 14 15; do
  src="$SRC/slide-$n.jpg"
  [ -f "$src" ] || { echo "missing $src" >&2; exit 1; }
  make_one "$src" "$OUT/slide-$n-480.jpg" 480 316
  make_one "$src" "$OUT/slide-$n-960.jpg" 960 632
  echo "slide-$n done"
done

echo
echo "Total: $(du -sh "$OUT" | cut -f1) in $OUT"
```

## Task 6: Pause toggle

**Files:**
- Create: `assets/js/rail.js`

WCAG 2.2.2 requires a pause mechanism for auto-moving content that runs longer than five seconds. Hover and focus pausing do not satisfy it — a touch user has no hover. This toggle is the reason the constant-drift design is shippable, not a nicety.

- [ ] **Step 1: Write the script**

Create `assets/js/rail.js`:

```js
/* UGCC homepage gallery rail — pause/resume toggle.
   The drift itself is pure CSS; this only flips a data attribute the
   stylesheet keys off. ES5, no dependencies, no build step.

   WCAG 2.2.2: auto-moving content that runs longer than 5s needs a pause
   mechanism available to every user, not just those who can hover. */
(function () {
  'use strict';

  function init() {
    var rails = document.querySelectorAll('.v2-rail');
    Array.prototype.forEach.call(rails, function (rail) {
      var btn = rail.querySelector('.v2-rail__toggle');
      if (!btn) return;

      btn.addEventListener('click', function () {
        var paused = rail.getAttribute('data-paused') === 'true';
        var next = !paused;
        rail.setAttribute('data-paused', next ? 'true' : 'false');
        btn.setAttribute('aria-pressed', next ? 'true' : 'false');
        btn.setAttribute('aria-label',
          next ? 'Resume the moving gallery' : 'Pause the moving gallery');
      });
    });
  }

  if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', init);
  } else {
    init();
  }
})();
```

- [ ] **Step 2: Verify the toggle**

Reload and run `tools/rail-check.js`.

Expected: `pause toggle controls the animation` → **PASS**, detail `running -> paused -> running; aria-pressed=true`.

By hand: click the button. The rail stops, the icon becomes a play triangle, and the drift resumes on a second click. Tab to the button with the keyboard and press Space — same behaviour, with a visible red focus ring.
